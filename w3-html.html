HTML Introduction

HTML is the standard markup language for creating Web pages.

What is HTML?

  • HTML stands for Hyper Text Markup Language
  • HTML is the standard markup language for creating Web pages
  • HTML describes the structure of a Web page
  • HTML consists of a series of elements
  • HTML elements tell the browser how to display the content
  • HTML elements label pieces of content such as "this is a heading", "this is a paragraph", "this is a link", etc.

A Simple HTML Document

<!DOCTYPE html>
<html>
<head>
<title>Page Title</title>
</head>
<body>

<h1>My First Heading</h1>
<p>My first paragraph.</p>

</body>
</html>

Example Explained

  • <!DOCTYPE html> defines that this document is an HTML5 document
  • <html> is the root element of an HTML page
  • <head> contains meta information about the HTML page
  • <title> specifies a title for the HTML page
  • <body> defines the document's body
  • <h1> defines a large heading
  • <p> defines a paragraph

What is an HTML Element?

An HTML element is defined by a start tag, some content, and an end tag:

<tagname> Content goes here... </tagname>

The HTML element is everything from the start tag to the end tag:

<h1>My First Heading</h1>
<p>My first paragraph.</p>
Start tagElement contentEnd tag
<h1>My First Heading</h1>
<p>My first paragraph.</p>
<br>nonenone

Note: Some HTML elements have no content (like the <br> element). These elements are called empty elements and do not have an end tag.

Web Browsers

The purpose of a web browser (Chrome, Edge, Firefox, Safari) is to read HTML documents and display them correctly.

A browser does not display the HTML tags, but uses them to determine how to display the document.

HTML Page Structure

<html>
<head>
<title>Page title</title>
</head>
<body>
<h1>This is a heading</h1>
<p>This is a paragraph.</p>
<p>This is another paragraph.</p>
</body>
</html>

Note: The content inside the <body> section will be displayed in a browser. The content inside the <title> element will be shown in the browser's title bar or tab.

HTML Editors

A simple text editor is all you need to learn HTML.

Learn HTML Using Notepad or TextEdit

Web pages can be created and modified by using professional HTML editors.

However, for learning HTML we recommend a simple text editor like Notepad (PC) or TextEdit (Mac).

We believe that using a simple text editor is a good way to learn HTML.

Follow the steps below to create your first web page with Notepad or TextEdit.

Step 1: Open Notepad (PC)

  • Windows 8 or later: Open the Start Screen (the window symbol at the bottom left on your screen). Type Notepad.
  • Windows 7 or earlier: Open Start > Programs > Accessories > Notepad

Step 1: Open TextEdit (Mac)

  • Open Finder > Applications > TextEdit
  • Change preferences to save files correctly:
    • In Preferences > Format > choose "Plain Text"
    • Under "Open and Save", check: "Display HTML files as HTML code instead of formatted text"
  • Then open a new document to place the code.

Step 2: Write Some HTML

Write or copy the following HTML code into Notepad or TextEdit:

<!DOCTYPE html>
<html>
<body>

<h1>My First Heading</h1>

<p>My first paragraph.</p>

</body>
</html>

Step 3: Save the HTML Page

Save the file on your computer. Select File > Save As in the Notepad or TextEdit menu.

  • Name the file index.htm
  • Set the encoding to UTF-8 (preferred for HTML files)

Tip: You can use either .htm or .html as file extension. There is no difference.

Step 4: View the HTML Page in Your Browser

Open the saved HTML file in your favorite browser (double click on the file, or right-click and choose "Open with").

HTML Basic Examples

In this chapter we will show some basic HTML examples.

Don't worry if we use tags you have not learned about yet.

HTML Documents

All HTML documents must start with a document type declaration: <!DOCTYPE html>.

The HTML document itself begins with <html> and ends with </html>.

The visible part of the HTML document is between <body> and </body>.

Example

<!DOCTYPE html>
<html>
<body>

<h1>My First Heading</h1>
<p>My first paragraph.</p>

</body>
</html>

The <!DOCTYPE> Declaration

The <!DOCTYPE> declaration represents the document type, and helps browsers to display web pages correctly.

  • It must only appear once, at the top of the page (before any HTML tags).
  • It is not case sensitive.
  • The HTML5 declaration is: <!DOCTYPE html>

HTML Headings

HTML headings are defined with the <h1> to <h6> tags.

<h1> defines the most important heading. <h6> defines the least important heading.

Example

<h1>This is heading 1</h1>
<h2>This is heading 2</h2>
<h3>This is heading 3</h3>

HTML Paragraphs

HTML paragraphs are defined with the <p> tag:

Example

<p>This is a paragraph.</p>
<p>This is another paragraph.</p>

HTML Links

HTML links are defined with the <a> tag:

Example

<a href="https://www.w3schools.com">This is a link</a>

The link's destination is specified in the href attribute.

Attributes are used to provide additional information about HTML elements.

You will learn more about attributes in a later chapter.

HTML Images

HTML images are defined with the <img> tag.

The source file (src), alternative text (alt), width, and height are provided as attributes:

Example

<img src="w3schools.jpg" alt="W3Schools.com" width="104" height="142">

HTML Elements

An HTML element is defined by a start tag, some content, and an end tag.

HTML Elements

The HTML element is everything from the start tag to the end tag:

<tagname>Content goes here...</tagname>

Examples of some HTML elements:

<h1>My First Heading</h1>
<p>My first paragraph.</p>
Start tagElement contentEnd tag

<p>My first paragraph.</p>

Note: Some HTML elements have no content (like the <br> element). These elements are called empty elements. Empty elements do not have an end tag!

Nested HTML Elements

HTML elements can be nested (this means that elements can contain other elements).

All HTML documents consist of nested HTML elements.

The following example contains four HTML elements (<html>, <body>, <h1> and <p>):

Example

<!DOCTYPE html>
<html>
<body>

<h1>My First Heading</h1>
<p>My first paragraph.</p>

</body>
</html>

Example Explained

  • The <html> element is the root element and it defines the whole HTML document.
  • It has a start tag <html> and an end tag </html>.
  • Inside the <html> element there is a <body> element.
  • The <body> element defines the document's body and contains other elements like <h1> and <p>.
  • The <h1> element defines a heading and has a start tag and end tag.
  • The <p> element defines a paragraph and also has a start tag and end tag.

Never Skip the End Tag

Some HTML elements will display correctly, even if you forget the end tag:

Example

<html>
<body>

<p>This is a paragraph
<p>This is a paragraph

</body>
</html>

However, never rely on this! Unexpected results and errors may occur if you forget the end tag!

Empty HTML Elements

HTML elements with no content are called empty elements.

The <br> tag defines a line break, and is an empty element without a closing tag:

Example

<p>This is a <br> paragraph with a line break.</p>

HTML is Not Case Sensitive

HTML tags are not case sensitive: <P> means the same as <p>.

The HTML standard does not require lowercase tags, but:

  • W3C recommends lowercase in HTML
  • Lowercase is required for stricter document types like XHTML

HTML Attributes

HTML attributes provide additional information about HTML elements and are always specified in the start tag.

  • All HTML elements can have attributes.
  • Attributes provide additional information about elements.
  • Attributes are always specified in the start tag.
  • Attributes usually come in name/value pairs like: name="value".

The href Attribute

The <a> tag defines a hyperlink. The href attribute specifies the URL:

<a href="https://www.w3schools.com">Visit W3Schools</a>

The src Attribute

The <img> tag uses the src attribute to embed an image:

<img src="img_girl.jpg">

Absolute vs Relative URLs

  • Absolute: src="https://www.w3schools.com/images/img_girl.jpg"
  • Relative: src="img_girl.jpg" or src="/images/img_girl.jpg"

The width and height Attributes

<img src="img_girl.jpg" width="500" height="600">

The alt Attribute

Alternate text for images:

<img src="img_typo.jpg" alt="Girl with a jacket">

The style Attribute

Used to apply inline CSS:

<p style="color:red;">This is a red paragraph.</p>

The lang Attribute

<html lang="en">
<html lang="en-US">

The title Attribute

Shows a tooltip when hovered:

<p title="I'm a tooltip">This is a paragraph.</p>

Best Practices

  • Always use lowercase attribute names: href, src, not HREF.
  • Always quote attribute values:
<a href="https://www.w3schools.com/html/">Good</a>
<a href=https://www.w3schools.com/html/>Bad</a>

Single or Double Quotes

<p title='John "ShotGun" Nelson'>
<p title="John 'ShotGun' Nelson">

HTML Headings

HTML headings are titles or subtitles that you want to display on a webpage.

HTML Heading Tags

HTML headings are defined with the <h1> to <h6> tags:

<h1>Heading 1</h1>
<h2>Heading 2</h2>
<h3>Heading 3</h3>
<h4>Heading 4</h4>
<h5>Heading 5</h5>
<h6>Heading 6</h6>

<h1> defines the most important heading, while <h6> defines the least important.

Heading Appearance

Browsers automatically add some margin before and after a heading to improve readability.

Why Headings Are Important

  • Search engines use headings to understand page structure.
  • Users scan headings to find relevant content quickly.
  • Use <h1> for main titles, followed by <h2>, and so on for sub-sections.
  • Do not use headings just to make text big or bold — use them to define content structure.

Custom Font Size

You can override the default size using CSS with the style attribute:

<h1 style="font-size:60px;">Heading 1</h1>

HTML <hr> - Horizontal Rule

The <hr> tag defines a thematic break and is usually displayed as a horizontal line.

Example

<h1>This is heading 1</h1>
<p>This is some text.</p>
<hr>
<h2>This is heading 2</h2>
<p>This is some other text.</p>
<hr>

<hr> is an empty tag, meaning it has no closing tag.

HTML <br> - Line Break

The <br> tag inserts a line break without starting a new paragraph.

Example

<p>This is<br>a paragraph<br>with line breaks.</p>

<br> is also an empty tag and does not need a closing tag.

The Poem Problem

This poem will not display correctly with just a <p> tag:

Incorrect:

<p>
  My Bonnie lies over the ocean.

  My Bonnie lies over the sea.

  My Bonnie lies over the ocean.

  Oh, bring back my Bonnie to me.
</p>

Solution - The HTML <pre> Element

The <pre> element preserves both spaces and line breaks, and uses a monospace font:

Correct:

<pre>
  My Bonnie lies over the ocean.

  My Bonnie lies over the sea.

  My Bonnie lies over the ocean.

  Oh, bring back my Bonnie to me.
</pre>

HTML Styles

The HTML style attribute is used to add styles to an element, such as color, font, size, and more.

Example

The HTML style Attribute

Setting the style of an HTML element can be done with the style attribute.

Syntax:

<tagname style="property:value;">

The property is a CSS property, and the value is a CSS value.

Background Color

The CSS background-color property defines the background color of an element.

Example 1: Set full page background

<body style="background-color:powderblue;">
  <h1>This is a heading</h1>

</body>

Example 2: Set background on individual elements

<h1 style="background-color:powderblue;">This is a heading</h1>
<p style="background-color:tomato;">This is a paragraph.</p>

Text Color

The CSS color property defines the text color:

<h1 style="color:blue;">This is a heading</h1>
<p style="color:red;">This is a paragraph.</p>

Fonts

The CSS font-family property defines the font to be used:

<h1 style="font-family:verdana;">This is a heading</h1>
<p style="font-family:courier;">This is a paragraph.</p>

Text Size

The CSS font-size property defines the size of the text:

<h1 style="font-size:300%;">This is a heading</h1>
<p style="font-size:160%;">This is a paragraph.</p>

Text Alignment

The CSS text-align property sets horizontal text alignment:

<h1 style="text-align:center;">Centered Heading</h1>
<p style="text-align:center;">Centered paragraph.</p>

HTML Comments

HTML comments are not displayed in the browser but can help document your code for yourself or others.

How to Write a Comment

Comments start with <!-- and end with -->.

<!-- This is a comment -->

Example

<!-- This is a comment -->
<p>This is a paragraph.</p>
<!-- Comments are ignored by the browser -->

This is a paragraph.

Why Use Comments?

  • To explain your code for future reference.
  • To make parts of the code easier to understand.
  • To hide code temporarily while testing or debugging.

Hide Elements with Comments

You can use comments to hide elements from being displayed in the browser:

<!-- <p>This will not be shown on the webpage.</p> -->

Nested Comments Are Not Allowed

HTML does not support nested comments:


<!--
  <!-- This will cause an error -->
-->

Note: Never nest one comment inside another — it will break the HTML parsing.

HTML Colors

HTML colors are used to set the background color, text color, border color, and more.

Using the style Attribute

The style attribute is used to apply color in HTML elements.

<h1 style="color:blue;">This is a blue heading</h1>

Color Names

You can use color names directly:

<p style="color:red;">Red Text</p>
<p style="background-color:lightgray;">Gray Background</p>

RGB Colors

Colors can be defined using RGB values:

<p style="color:rgb(255,0,0);">Red using RGB</p>

Hexadecimal Colors

Colors can be defined using hex codes:

<p style="color:#ff0000;">Red using HEX</p>

HSL Colors

HSL (Hue, Saturation, Lightness) format is also supported:

<p style="color:hsl(0, 100%, 50%);">Red using HSL</p>

Transparent Colors (RGBA & HSLA)

RGBA and HSLA allow you to set opacity using the alpha value (0 to 1):

<p style="color:rgba(255,0,0,0.5);">Red with 50% opacity</p>
<p style="color:hsla(0, 100%, 50%, 0.3);">Red with 30% opacity</p>

Background Color Example

  </body>

HTML CSS

CSS stands for Cascading Style Sheets. CSS describes how HTML elements are to be displayed on screen, paper, or in other media.

Why Use CSS?

  • CSS saves a lot of work by controlling the layout of multiple web pages all at once.
  • It allows you to separate content from design.
  • It improves page load speed and maintenance.

Three Ways to Apply CSS

  1. Inline CSS – by using the style attribute inside HTML elements.
  2. Internal CSS – by placing a <style> block in the <head> section of the document.
  3. External CSS – by linking an external .css file using <link>.

1. Inline CSS Example

<p style="color:blue; font-size:18px;">This is a blue paragraph.</p>

2. Internal CSS Example

<head>
  <style>
    body {
      background-color: lightyellow;
    }
    h1 {
      color: navy;
    }
    p {
      color: darkgreen;
    }
  </style>
</head>

3. External CSS Example

<head>
  <link rel="stylesheet" href="styles.css">
</head>

And in styles.css:

body {
  background-color: #f0f0f0;
}
h1 {
  color: blue;
}
p {
  font-size: 16px;
}

CSS Syntax

A CSS rule consists of a selector and a declaration block.

p {
  color: red;
  font-size: 20px;
}
  • Selector – selects the HTML element.
  • Property – the style you want to change (e.g., color).
  • Value – the new value for the property (e.g., red).

HTML Links

HTML links are hyperlinks. You can click on a link and jump to another document.

The <a> Tag

HTML links are defined with the <a> tag. The most important attribute of the <a> element is the href attribute, which indicates the link's destination.

Example:

<a href="https://www.w3schools.com">Visit W3Schools</a>

The target Attribute

Use the target attribute to define where to open the linked document:

  • _self – Default. Opens in the same tab.
  • _blank – Opens in a new tab or window.
  • _parent – Opens in the parent frame.
  • _top – Opens in the full body of the window.

Example (open in new tab):

<a href="https://www.google.com" target="_blank">Open Google</a>

Using an Image as a Link

<a href="https://www.w3schools.com">
  <img src="logo.png" alt="W3Schools" width="100" height="100">
</a>

Linking to an Email Address

<a href="mailto:someone@example.com">Send Email</a>

Button Styled Link

<a href="https://example.com" style="padding: 10px 20px; background: blue; color: white; text-decoration: none; border-radius: 5px;">
  Click Me
</a>

Absolute vs. Relative URLs

  • Absolute URL: href="https://www.example.com/page"
  • Relative URL: href="/page" or href="page.html"

Best Practices

  • Always use https:// for secure links.
  • Use descriptive link text (e.g., "Read our HTML Guide") instead of "Click here".
  • For accessibility, ensure link text is clear out of context.

HTML Favicon

A favicon is a small icon that appears in the browser tab, next to the page title. It helps identify your website visually.

How to Add a Favicon

Use the <link> element inside the <head> section of your HTML document:

<link rel="icon" type="image/x-icon" href="favicon.ico">

Example:

<!DOCTYPE html>
<html>
<head>
  <title>My Website</title>
  <link rel="icon" type="image/x-icon" href="favicon.ico">
</head>
<body>

  <h1>Welcome to My Website</h1>

</body>
</html>

Favicon File Format

  • .ico – Most widely supported (recommended)
  • .png, .gif, .jpg – Also work in modern browsers
  • Recommended size: 16×16 or 32×32 pixels

Tips

  • Place favicon.ico in the root directory for automatic detection.
  • Always define it explicitly with the <link> tag for reliability.

Why Use a Favicon?

  • Improves user experience and branding.
  • Makes your site easier to recognize in browser tabs and bookmarks.

HTML Tables

The <table> element in HTML is used to display tabular data using rows and columns.

Basic Table Example

<table border="1">
  <tr>
    <th>Name</th>
    <th>Age</th>
  </tr>

    <td>Alice</td>
    <td>24</td>

Table Tags Overview

  • <th>: Defines a header cell (bold and centered)

Adding a Caption

You can add a title to your table using <caption>:

<table border="1">
  <caption>Student Info</caption>
  <tr>
    <th>Name</th>

</table>

Colspan and Rowspan

These attributes allow cells to span multiple columns or rows.

<table border="1">
  <tr>

  </tr>
  <tr>
    <td>Alice</td>
    <td>Smith</td>
  </tr>

HTML Block & Inline Elements

Every HTML element has a default display value. The two most common values are block and inline.

Block-level Elements

Block elements always start on a new line and take up the full width available.

Examples of block-level elements:

  • <div>
  • <p>
  • <h1> to <h6>
  • <form>
  • <header>
  • <footer>
  • <section>

Example:

<p>This is a block element</p>
<div>This is another block element</div>

Inline Elements

Inline elements do not start on a new line and only take up as much width as necessary.

Examples of inline elements:

  • <span>
  • <a>
  • <img>
  • <strong>
  • <em>
  • <br>

Example:

<p>This is <span>inline</span> text.</p>

Changing Display Type

You can use CSS to change the default display behavior of elements.

<style>
  span {
    display: block;
  }
</style>

This will make all <span> elements behave like block elements.

HTML <div> Element

The <div> tag defines a division or a section in an HTML document.

The <div> element is often used as a container for other HTML elements to style them with CSS or to perform certain tasks with JavaScript.

Example

  <h1>This is a heading</h1>
  <p>This is a paragraph.</p>
</div>

Why Use <div>?

  • To group related elements.
  • To apply common CSS styles.
  • To create layout structures.

Styling Divs

Use CSS to style <div> elements and control their size, position, background, and more.

<style>
  .container {
    width: 80%;
    margin: auto;
    background-color: #f4f4f4;
    padding: 20px;
  }

<div class="container">

</div>

You can nest <div> elements inside each other for complex layouts:

<div style="background-color:lightgray;">

    <h2>Nested div</h2>

  </div>

The class attribute is used to define equal styles for elements with the same class name.

It allows you to apply CSS styles to multiple elements without repeating the style code.

The id attribute is used to uniquely identify a specific HTML element.

HTML Iframes

An iframe (short for inline frame) is used to embed another document within the current HTML document.

Basic Syntax

<iframe src="https://www.w3schools.com" width="600" height="400"></iframe>

This will embed the W3Schools homepage into your webpage.

Iframe - Set Height and Width

Use the width and height attributes to specify the size:

<iframe src="demo.html" width="800" height="600"></iframe>

Iframe - Remove Border

Use the style attribute with border:none; to remove the border:

<iframe src="demo.html" style="border:none;" width="800" height="600"></iframe>

You can target an iframe with a hyperlink:

<a href="page2.html" target="myframe">Open in iframe</a>

Some websites prevent their content from being loaded into iframes for security reasons (using the X-Frame-Options HTTP header).

Iframe Attributes

  • width and height – Size of the iframe
  • name – Assigns a name to the iframe (used by target)
  • style – CSS styling

Note

Using iframes can affect performance and SEO. Use them only when necessary.

HTML Input Attributes

HTML <input> elements support various attributes that define behavior, validation, and user interaction.

Common Input Attributes

  • type – Specifies the input type (text, password, email, etc.)
  • name – Name of the input (used in form submission)
  • value – Predefined value for the input
  • placeholder – Grey hint text shown when input is empty
  • required – Makes the input mandatory before form submission
  • readonly – Makes the input non-editable but submittable
  • disabled – Disables the input (not editable or submittable)
  • maxlength – Sets the max number of characters allowed
  • min and max – Defines a numeric or date range
  • step – Specifies the legal number intervals (used with number/date)
  • autofocus – Automatically focuses the input when the page loads
  • autocomplete – Suggests previously entered values
  • pattern – Regex pattern for validating the input
  • multiple – Allows selecting multiple files/emails
  • size – Visual width of the input in characters
  • minlength – Minimum number of characters

Example

<form>
  Username: <input type="text" name="username" required maxlength="20"><br>
  Email: <input type="email" name="email" placeholder="Enter your email" required><br>
  Password: <input type="password" name="pwd" minlength="6"><br>
  Age: <input type="number" min="18" max="99"><br>
  Upload File: <input type="file" multiple><br>
  <input type="submit">
</form>

Note:

  • readonly allows form submission; disabled does not.
  • pattern is powerful for custom input formats like phone numbers or postal codes.
  • Use autocomplete="off" if you want to prevent browsers from remembering user input.

HTML Input Form Attributes

HTML provides several special attributes that can be used on <input> elements to control how they behave inside or outside forms.

Key Form-Related Attributes for Inputs

  • form: Specifies the id of the form the input belongs to (even if it’s outside the <form>).
  • formaction: URL where the form data is submitted (only used with <input type="submit"> or <button type="submit">).
  • formenctype: Specifies how the form data should be encoded (e.g., multipart/form-data for file uploads).
  • formmethod: Defines the HTTP method used when submitting the form (e.g., get, post).
  • formtarget: Specifies where to display the response (e.g., _self, _blank).
  • formnovalidate: Disables form validation on submit.

Example – Input Outside the Form

<form id="myForm" action="/submit">
  <input type="text" name="name" required>
</form>

<input type="submit" value="Submit" form="myForm">

Example – Overriding Form Defaults

<form action="/default" method="post">
  <input type="submit" value="Default Submit">

  <input type="submit" value="Submit with GET" formmethod="get" formaction="/custom">
</form>

Common Use Cases

  • Use form to link inputs to forms when they are placed outside the form tag (e.g., in toolbars).
  • Use formaction and formmethod to have multiple submit buttons in one form that perform different actions.
  • Use formnovalidate for "preview" buttons that shouldn’t validate data.

Note:

The form* attributes override the values set on the <form> tag for that specific input/button only.

HTML Web APIs

Web APIs (Application Programming Interfaces) provide browser-based JavaScript interfaces for interacting with browser features and the user’s environment.

Commonly Used Web APIs

  • DOM API: Manipulate HTML and CSS content on the page.
  • Geolocation API: Get the user’s location (with permission).
  • Fetch API: Request data from servers (replacement for XMLHttpRequest).
  • Canvas API: Draw graphics and animations via JavaScript.
  • Drag and Drop API: Enable draggable UI elements.
  • Web Storage API: Store data in the browser (localStorage and sessionStorage).
  • Notifications API: Show desktop notifications.

Example: Geolocation API

Use JavaScript to get the user’s latitude and longitude:

<button onclick="getLocation()">Get Location</button>
<p id="demo"></p>

<script>
function getLocation() {
  if (navigator.geolocation) {
    navigator.geolocation.getCurrentPosition(function(position) {
      document.getElementById("demo").innerHTML =
        "Latitude: " + position.coords.latitude +
        "<br>Longitude: " + position.coords.longitude;
    });
  } else {
    document.getElementById("demo").innerHTML = "Geolocation is not supported by this browser.";
  }
}
</script>

Example: localStorage API

Store and retrieve data in the browser:

// Store data
localStorage.setItem("username", "Shrinivas");

// Retrieve data
const user = localStorage.getItem("username");
console.log(user); // Output: Shrinivas

Example: Fetch API

Fetch data from an API or backend:

fetch('https://jsonplaceholder.typicode.com/posts/1')
  .then(response => response.json())
  .then(data => console.log(data));

Other Web APIs

  • Clipboard API: Copy/paste text via JavaScript.
  • Speech API: Convert speech to text or vice versa.
  • Battery API: Check the battery level of the user’s device.
  • Vibration API: Control mobile device vibration.

These APIs allow developers to build interactive, dynamic, and user-aware websites and applications.

HTML Geolocation

The HTML Geolocation API is used to get the geographical position of a user. It is a part of the browser's Web APIs and is available in JavaScript.

Browser Support

Most modern browsers support the Geolocation API, but the user must allow location access when prompted.

Basic Example

Click the button below to get your current position:

Geolocation Methods

  • navigator.geolocation.getCurrentPosition(): Get the user's current location once.
  • navigator.geolocation.watchPosition(): Track the user's location continuously.
  • navigator.geolocation.clearWatch(): Stop watching the location.

You can pass an options object to customize the accuracy and timeout:

  • Delivery tracking
  • Local weather or news updates

Note: The Geolocation API only works on secure contexts (HTTPS) or localhost.
